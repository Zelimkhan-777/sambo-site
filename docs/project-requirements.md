# Project Requirements

## О проекте

Многостраничный сайт Федерации самбо Чеченской Республики.

Основные задачи:

- представление федерации;
- публикация новостей;
- размещение документов;
- представление руководства и тренеров;
- публикация информации о грантах;
- предоставление контактов.

## Визуальное направление

Строгий современный спортивный сайт.

Основная палитра:

- чёрный;
- белый;
- графитовый;
- светло-серый;
- редкие красные и синие акценты.

Избегать:

- неона;
- glassmorphism;
- чрезмерных градиентов;
- одинаковых округлых карточек;
- дизайна фитнес-клуба.

## Страницы

- `/` — главная;
- `/about` — о федерации;
- `/documents` — документы;
- `/grants` — гранты;
- `/news` — новости;
- `/news/:slug` — отдельная новость;
- `/coaches` — тренеры;
- `*` — 404.

## Главная

Секции:

- hero;
- кратко о федерации;
- направления деятельности;
- последние новости;
- руководство и тренеры;
- контактный блок.

Статистику показывать только при наличии подтверждённых данных.

## О Федерации

Содержимое:

- официальное название;
- описание деятельности;
- цели и задачи;
- руководство;
- направления работы;
- контакты.

Не придумывать историю и даты.

## Новости

Необходимо:

- список новостей;
- фильтрация;
- поиск;
- сортировка;
- страница отдельной новости;
- похожие материалы;
- копирование ссылки.

## Тренеры

Показывать:

- ФИО;
- должность;
- направление;
- фото или placeholder;
- только подтверждённые достижения.

## Документы

Необходимо:

- каталог;
- поиск;
- категории;
- фильтр по году;
- просмотр и скачивание доступных файлов;
- empty state при отсутствии документов.

## Гранты

При наличии данных показывать программы и статусы.

Если данных нет, использовать empty state и не создавать вымышленные карточки.

## Контент

Использовать данные из официальных и подтверждённых источников.

Не придумывать:

- имена;
- должности;
- даты;
- достижения;
- соревнования;
- документы;
- статистику;
- гранты;
- суммы.

## Изображения

Использовать локальные изображения и подтверждённые материалы.

Не использовать случайные фотографии без проверки.

Если качественного фото нет:

- использовать placeholder;
- уменьшать формат изображения;
- применять кадрирование;
- не растягивать слабые фотографии в hero.

## Адаптивность

Сайт должен корректно работать от 320 px.

Особенно проверить:

- Header;
- hero;
- карточки;
- фильтры;
- документы;
- footer;
- отсутствие горизонтального скролла.

## MVP готов, когда

- работают все маршруты;
- готова главная;
- работают темы;
- работает мобильное меню;
- реализованы новости;
- реализованы тренеры;
- есть документы и гранты;
- нет ошибок сборки;
- нет вымышленных фактов.
